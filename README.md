🎮 โครงสร้างและสถาปัตยกรรมของแอป Story Circle 

storycircle.up.railway.app

Story Circle เป็นเว็บแอปพลิเคชันรูปแบบ Monorepo ที่เป็นเกมกระดานวงเหล้าแบบเรียลไทม์ (Real-time Multiplayer Board Game) เล่นพร้อมกันผ่านเบราว์เซอร์บนมือถือ โดยเน้นธีม Neon Dark Party

📁 โครงสร้างโปรเจกต์ (Project Structure)
โปรเจกต์นี้แบ่งออกเป็น 2 ส่วนหลัก:

1. Backend (backend/)
ทำหน้าที่เป็น API server และ Real-time websocket gateway:

prisma/schema.prisma: ออกแบบโมเดลฐานข้อมูล PostgreSQL ด้วย Prisma ORM
src/index.js: จุดเชื่อมต่อเซิร์ฟเวอร์หลัก ตั้งค่า Express, CORS, HTTP server และ Socket.IO
src/routes/: จัดการเส้นทาง REST API เช่น ระบบลงทะเบียน/เข้าสู่ระบบ และการดึงข้อมูลสถิติ
src/socket/: ควบคุม logic การเล่นแบบเรียลไทม์ผ่าน Socket.IO โดยแยกเป็นโมดูลย่อยตามเกม
src/utils/: เครื่องมือช่วยในการคำนวณและสุ่มผล เช่น horseEngine.js สำหรับจำลองการวิ่งของม้า


2. Frontend (frontend/)
หน้าเว็บแอปพลิเคชันไคลเอนต์ พัฒนาด้วย React:

src/App.jsx: ระบบกำหนดเส้นทาง (Routing) ประกอบด้วยหน้า Landing, Login, Register, JoinRoom, Lobby, Room, Game, และ Stats
src/context/: สถานะโกลบอล (Global State) เช่น AuthContext (ระบบล็อกอิน) และ SocketContext (การเชื่อมต่อ Socket.IO)
src/components/: คอมโพเนนต์ย่อยที่นำกลับมาใช้ใหม่ เช่น ตัวจับเวลา (Timer), ป้ายข้อมูลม้า (HorseCard), รายชื่อผู้เล่น (PlayerList), และตัวเลือกโหวต (VoteOption)
src/pages/: หน้าหลักของการเล่นเกมแต่ละโหมด
src/index.css: จัดการสไตล์ Neon Dark ธีมปาร์ตี้กลางคืนทั้งหมด (ใช้ Vanilla CSS)


🛠️ เทคโนโลยีสแต็ก (Tech Stack)
ส่วนงาน (Layer)	เทคโนโลยีที่ใช้ (Technologies)	รายละเอียด (Details)
Frontend	React (Vite)	สปาเฟรมเวิร์กที่โหลดได้รวดเร็วและตอบสนองทันที
Styling	Vanilla CSS	ใช้ CSS ดิบร่วมกับ custom tokens เพื่อสร้าง Neon Dark Theme
Backend	Node.js (Express)	เว็บเซิร์ฟเวอร์น้ำหนักเบาสำหรับ REST API
Real-time	Socket.IO	การเชื่อมต่อ WebSockets สองทิศทางสำหรับการเล่นแบบเรียลไทม์
Database	PostgreSQL + Prisma ORM	เก็บข้อมูลผู้ใช้ ห้องประวัติการเล่น คะแนน และรายละเอียดเกม
AI Integration	OpenRouter (Gemini-2.5-Flash)	ใช้ช่วยสุ่มคิดคำถามปั่นๆ ในเกม Minority Vote


🔑 ฟีเจอร์และกลไกการทำงานหลัก (Core Features)


👥 1. ระบบผู้ใช้ชั่วคราวและการเข้าร่วมแบบเร่งด่วน (Guest Login & Quick Join)
Host Account: มีเพียงโฮสต์ (เจ้าของห้อง) เท่านั้นที่ต้องลงทะเบียนและเข้าสู่ระบบด้วยชื่อผู้ใช้และรหัสผ่านจริง
Guest Account: เพื่อนคนอื่นๆ สามารถแชร์ลิงก์ตรงหรือสแกน QR Code จากหน้าล็อบบี้ พิมพ์เฉพาะชื่อเล่นและสุ่มตัวอวาตาร์ ระบบจะสร้างบัญชีผู้ใช้ชั่วคราวที่มีรูปแบบ username#guest อัตโนมัติและสิทธิ์เข้าร่วมเล่นทันที


🔄 2. การซิงค์และสลับเกมกลางห้อง (Game Swapping & Sync Lobby)
โฮสต์มีอำนาจในการสลับประเภทเกมระหว่าง แข่งม้า (Horse Race), เสียงส่วนน้อย (Minority Vote), หรือ ร่มชูชีพวัดใจ (Panic Jump) ได้ตลอดเวลาโดยไม่ต้องปิดห้องสร้างใหม่
เมื่อสิ้นสุดเกม โฮสต์กด "กลับล็อบบี้" ระบบจะส่งคำสั่ง Socket ไปยังไคลเอนต์ทุกคนและดึงหน้าจอทุกคนกลับล็อบบี้ทันที


🧹 3. ระบบล้างฐานข้อมูลอัตโนมัติ (Zero-History Database Cleanups)
เพื่อป้องกันปัญหาระบบฐานข้อมูลทำงานหนักจากรอบเกมเก่า ระบบจะทำลายข้อมูลการแทงพนันหรือการโหวตของรอบก่อนหน้าทิ้งทันทีเมื่อกดเริ่มรอบใหม่
เมื่อผู้เล่นคนสุดท้ายออกจากห้อง ระบบจะทำการลบห้องนั้นออกจากตารางห้องแบบ Cascade (ทำให้ข้อมูลเกม ม้า และการแทงในห้องนั้นถูกลบไปด้วย) พร้อมทั้งลบชื่อบัญชี Guest ที่ไม่เคลื่อนไหวออกด้วย


🎮 เจาะลึกกลไกทั้ง 3 เกม (Game Details)
1. 🏇 เกมแข่งม้า (Horse Race)
การวิเคราะห์ดาวของกูรู: ม้าแข่งมี 8 ตัว แต่ละตัวมีค่าพลังลับ (speed, stamina, luck) ก่อนการแข่งจะปิดบังพลังจริง แต่จะให้ 3 กูรู (กรู(ไม่)รู, กรูเดา, กรูมั่ว) ช่วยวิเคราะห์เป็นระดับดาว (1-5 ดาว)
กฎเกณฑ์การประเมิน: คำนวณผ่าน Seed ของรอบการเล่น ทุกกูรูจะมีม้าเต็ง 5 ดาว 👑 เสมอ 1 ตัว และม้าบ๊วย 1 ดาว 💩 เสมอ 1 ตัว
การสุ่มการวิ่ง: ดำเนินการผ่านเครื่องคำนวณ (horseEngine.js) และส่งพิกัดการเคลื่อนที่ของม้าแบบเรียลไทม์เพื่อแสดงภาพแอนิเมชันการวิ่ง
การตัดสิน:
แทงม้าอันดับ 1-3 $\rightarrow$ รอด (ได้ดาวสะสม ⭐ สำหรับเก็บสถิติ)
แทงม้าอันดับ 4-8 $\rightarrow$ ดื่ม! (เพิ่มจำนวนการดื่มในหน้าสถิติ 🍺)


3. 🗳️ เกมเสียงส่วนน้อย (Minority Vote)
คำถาม AI: ใช้ API ของ OpenRouter เชื่อมต่อกับ Gemini 2.5 Flash ในการส่งคำสั่ง (System Prompt) เพื่อให้ AI ตอบกลับคำถามวงเหล้าภาษาไทยปั่นๆ 2 ตัวเลือกในรูปแบบ JSON เสมอ หากอินเทอร์เน็ตมีปัญหาระบบจะสลับไปดึงคำถามออฟไลน์ในตัวเซิร์ฟเวอร์
การโหวตแบบลับ: ผู้เล่นทุกคนกดโหวตตัวเลือก A หรือ B โดยคนในห้องจะไม่เห็นคะแนนโหวตจนกว่าจะหมดเวลา
การตัดสิน: ฝั่งที่เลือกช้อยส์ที่มีคนเลือก น้อยกว่า จะต้อง ดื่ม! 🍺 (กรณีแต้มเท่ากัน รอดหรือโดนทุกคนแล้วแต่ตกลงกัน)
UI พิเศษ: แผนภูมิแท่งสรุปสถิตินีออนสามารถใช้นิ้วเลื่อนสไลเดอร์ซูมกราฟได้ 1x - 3x เพื่อให้อ่านชื่อผู้เล่นแต่ละข้างได้สะดวกบนจอมือถือ


5. 🪂 เกมร่มชูชีพวัดใจ (Panic Jump)
การเอาชีวิตรอดจำกัด: ระบบจะสุ่มโควต้ายอดผู้รอดชีวิต (เช่น 1-3 คนจากผู้เล่นทั้งหมด) ทำให้ไม่มีใครรู้ล่วงหน้าว่ารอบนั้นมีคนรอดได้กี่คน
ความเสี่ยงและแต้ม: มีเวลา 8 วินาที ยิ่งกดโดดช้า (ทนรอใกล้ขีดจำกัด) ยิ่งได้รับตัวคูณแต้มเยอะ แต่ถ้ากะจังหวะช้าจนโควต้าเต็มก่อน หรือรอจนหมดเวลา (ดิ่งพื้นดิน) $\rightarrow$ จะตายทันที
แต้มแอนิเมชัน: ทุกคนจะดิ่งพสุธาพร้อมกัน หากร่มชูชีพกางได้ทันจะรอดสะสมแต้ม หากตกลงมาตายจะมีแอนิเมชันไฟลุกและถูกลงโทษดื่ม!
การสิ้นสุด: เล่น 1-10 รอบตามที่โฮสต์ตั้งไว้ คนที่ได้แต้มรวมต่ำที่สุดในตอนท้ายจะต้องดื่มบทลงโทษเพียวชุดใหญ่!
